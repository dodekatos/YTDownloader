V0.5.0

(Current)

  • Significant dependency updater improvements:
    • Added automatic updates for dependencies. It will only run once a week after Firefox starts, updating only the dependencies with updates available. By default, this will be enabled because that's really how it should've been from the start. But if you're used to manually updating the dependencies, this can be disabled in the Settings.
      Note that reinstalling the addon will cause the opt out preference to be cleared.
    • You might see a notification about the addon requiring a new permission for "storage".
      This permission only allows this addon to access its own sandboxed storage API. This storage is only used to store the last time an automatic dependency update was ran, and whether you are opted in to see update notes whenever the addon updates.
    • Significantly improved the speed of manual dependency updates in the Settings by allowing multiple dependency update checks to be run simultaneously

  • Significant Re-encode Tool improvements:
    • Added GPU acceleration support for H264/H265/AV1. Nvidia works, but Intel and AMD are currently untested, but should work
    • Added H264 video codec
    • Added notice for 10min+ ETAs to recommend using GPU acceleration
    • Improved the accuracy of the rough ETA
    • Removed the experimental 'Attach Thumbnail' option due to it not working. If you have any output files that don't have a thumbnail (usually just MKV+AV1), open the output file in the Windows Photos app to generate a thumbnail

  • Additional improvements:
    • Improved debug logging to prevent a rare issue with the file growing too large due to it sometimes being unable to truncate the file
    • Added icons to the addon's tabs (I'll change the icon to something less placeholder-y in a future update)
    • Changed the storage API used for the update notes toggle to storage.local so if you've ticked this before, you'll probably have to tick it again
